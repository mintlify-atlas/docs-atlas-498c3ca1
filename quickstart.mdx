# Quickstart

This guide will help you make your first request with Better Fetch and explore its key features.

## Basic Request

<Steps>
  <Step title="Import Better Fetch">
    Start by importing the `betterFetch` function:
    
    ```typescript
    import { betterFetch } from "@better-fetch/fetch";
    ```
  </Step>

  <Step title="Make Your First Request">
    Make a simple GET request and handle the response:
    
    ```typescript
    const { data, error } = await betterFetch("https://api.github.com/users/octocat");
    
    if (error) {
      console.error("Request failed:", error.status, error.statusText);
      return;
    }
    
    console.log("User data:", data);
    ```
    
    <Note>
      Notice the `{ data, error }` pattern - Better Fetch returns errors as values by default, no try/catch required!
    </Note>
  </Step>

  <Step title="Add Type Safety with Schema Validation">
    Install a schema validator (we'll use Zod):
    
    ```bash
    npm install zod
    ```
    
    Then add schema validation to get full type safety:
    
    ```typescript
    import { betterFetch } from "@better-fetch/fetch";
    import { z } from "zod";
    
    const userSchema = z.object({
      login: z.string(),
      name: z.string(),
      avatar_url: z.string(),
      bio: z.string().nullable(),
    });
    
    const { data, error } = await betterFetch("https://api.github.com/users/octocat", {
      output: userSchema,
    });
    
    if (error) {
      console.error("Request failed:", error);
      return;
    }
    
    // data is now fully typed!
    console.log(data.login); // TypeScript knows this is a string
    ```
    
    <Tip>
      The `output` option validates the response and provides automatic type inference!
    </Tip>
  </Step>
</Steps>

## POST Request with Validation

Here's a complete example showing a POST request with both input and output validation:

```typescript
import { betterFetch } from "@better-fetch/fetch";
import { z } from "zod";

// Define schemas
const createUserSchema = z.object({
  name: z.string(),
  email: z.string().email(),
});

const userResponseSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string(),
  createdAt: z.string(),
});

// Make the request
const { data, error } = await betterFetch("https://api.example.com/users", {
  method: "POST",
  body: {
    name: "John Doe",
    email: "john@example.com",
  },
  output: userResponseSchema,
});

if (error) {
  console.error("Failed to create user:", error);
  return;
}

console.log("Created user:", data.id, data.name);
```

## Error Handling Patterns

Better Fetch offers flexible error handling:

<Tabs>
  <Tab title="Errors as Values (Default)">
    The default pattern returns errors as values:
    
    ```typescript
    const { data, error } = await betterFetch("https://api.example.com/data");
    
    if (error) {
      // Handle error
      if (error.status === 404) {
        console.log("Resource not found");
      } else if (error.status >= 500) {
        console.log("Server error");
      }
      return;
    }
    
    // Use data safely
    console.log(data);
    ```
  </Tab>

  <Tab title="Throw on Error">
    If you prefer exceptions, use `throw: true`:
    
    ```typescript
    try {
      const data = await betterFetch("https://api.example.com/data", {
        throw: true,
      });
      
      console.log(data);
    } catch (error) {
      console.error("Request failed:", error);
    }
    ```
  </Tab>
</Tabs>

## Configure Headers and Auth

Add authentication and custom headers:

```typescript
const { data, error } = await betterFetch("https://api.example.com/protected", {
  headers: {
    "content-type": "application/json",
  },
  auth: {
    type: "Bearer",
    token: "your-token-here",
  },
});
```

## Create a Configured Client

For multiple requests to the same API, create a configured client:

```typescript
import { createFetch } from "@better-fetch/fetch";

const api = createFetch({
  baseURL: "https://api.example.com",
  headers: {
    "content-type": "application/json",
  },
  auth: {
    type: "Bearer",
    token: process.env.API_TOKEN,
  },
});

// Now make requests with the configured client
const { data, error } = await api("/users");
```

## Add Retry Logic

Configure automatic retries for failed requests:

```typescript
const { data, error } = await betterFetch("https://api.example.com/data", {
  retry: {
    attempts: 3,
    delay: 1000,
    methods: ["GET", "POST"],
  },
});
```

## Install Better Fetch

Install Better Fetch using your preferred package manager:

<CodeGroup>

```bash npm
npm install @better-fetch/fetch
```

```bash yarn
yarn add @better-fetch/fetch
```

```bash pnpm
pnpm add @better-fetch/fetch
```

```bash bun
bun add @better-fetch/fetch
```

</CodeGroup>

## TypeScript Setup

Better Fetch is built with TypeScript and provides full type safety out of the box. No additional configuration is required.

<Note>
  **TypeScript version**: Better Fetch works best with TypeScript 5.0 or higher for optimal type inference.
</Note>

## Schema Validation (Optional)

Better Fetch supports schema validation using the [Standard Schema](https://github.com/standard-schema/standard-schema) specification. Install your preferred validation library:

<Tabs>
  <Tab title="Zod">
    ```bash
    npm install zod
    ```
    
    Zod is the most popular TypeScript-first schema validation library.
  </Tab>
  
  <Tab title="Valibot">
    ```bash
    npm install valibot
    ```
    
    Valibot is a lightweight alternative with excellent bundle size.
  </Tab>
  
  <Tab title="ArkType">
    ```bash
    npm install arktype
    ```
    
    ArkType provides runtime type validation with TypeScript syntax.
  </Tab>
</Tabs>

<Tip>
  You can use Better Fetch without any validation library. Schema validation is completely optional and only needed when you want to validate request/response data.
</Tip>

## Verify Installation

Create a simple test file to verify your installation:

```typescript test.ts
import { betterFetch } from "@better-fetch/fetch";

const { data, error } = await betterFetch("https://jsonplaceholder.typicode.com/todos/1");

if (error) {
  console.error("Request failed:", error.status);
} else {
  console.log("Success:", data);
}
```

Run it with your runtime:

<CodeGroup>

```bash Node.js
node test.ts
```

```bash Deno
deno run --allow-net test.ts
```

```bash Bun
bun run test.ts
```

</CodeGroup>

If you see the response data logged, you're ready to go!

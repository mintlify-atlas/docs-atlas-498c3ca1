# Better Fetch

Better Fetch is a modern, type-safe fetch wrapper for TypeScript that works seamlessly across browsers, Node.js (v18+), workers, Deno, and Bun. It enhances the native Fetch API with powerful features like schema validation, plugins, hooks, and built-in error handling.

## Key Features

Better Fetch provides everything you need for modern HTTP requests:

- **Type-safe fetch wrapper** with full TypeScript support
- **Schema validation** using Standard Schema (Zod, Valibot, ArkType)
- **Plugin system** for extending functionality
- **Hook system** for request/response lifecycle (onRequest, onResponse, onSuccess, onError, onRetry)
- **Retry mechanism** with configurable strategies
- **Error handling as values** (non-throwing by default)
- **Authorization helpers** and header management
- **Dynamic URL parameter** substitution
- **Timeout support** with automatic cancellation
- **Custom fetch implementations** for testing and mocking

## Quick Example

```typescript
import { betterFetch } from "@better-fetch/fetch";
import { z } from "zod";

const { data, error } = await betterFetch("https://api.example.com/users/1", {
  output: z.object({
    id: z.number(),
    name: z.string(),
    email: z.string().email(),
  }),
});

if (error) {
  console.error("Request failed:", error.status, error.statusText);
  return;
}

// data is fully typed based on the schema
console.log(data.name); // TypeScript knows this is a string
```

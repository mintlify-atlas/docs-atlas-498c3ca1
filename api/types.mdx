## Core Types

### BetterFetchOption

The main options type for configuring requests.

```typescript
type BetterFetchOption<
  Body = any,
  Query extends Record<string, any> = any,
  Params extends Record<string, any> | Array<string> | undefined = any,
  Res = any,
  ExtraOptions extends Record<string, any> = {},
  Headers extends Record<string, string> = {},
> = ExtraOptions &
  Omit<RequestInit, "body" | "headers"> &
  FetchHooks<Res> & {
    // Request Configuration
    timeout?: number
    customFetchImpl?: FetchEsque
    plugins?: BetterFetchPlugin[]
    baseURL?: string
    throw?: boolean
    
    // Authentication
    auth?: Auth
    
    // Request Data
    headers?: CustomHeaders<Headers> | Headers
    body?: Body
    query?: Query
    params?: Params
    method?: StringLiteralUnion<PayloadMethod | NonPayloadMethod>
    
    // Advanced Options
    duplex?: "full" | "half"
    jsonParser?: (text: string) => Promise<any> | any
    
    // Retry Configuration
    retry?: RetryOptions
    retryAttempt?: number
    
    // Schema Validation
    output?: StandardSchemaV1 | typeof Blob | typeof File
    errorSchema?: StandardSchemaV1
    disableValidation?: boolean
    
    // Abort Control
    signal?: AbortSignal | null
  }
```

### BetterFetchResponse

The response type returned by `betterFetch`.

```typescript
type BetterFetchResponse<
  T,
  E extends Record<string, unknown> | unknown = unknown,
  Throw extends boolean = false,
> = Throw extends true ? T : Data<T> | Error<E>

type Data<T> = { data: T; error: null }

type Error<E> = {
  data: null
  error: (E extends Record<string, any> ? E : { message?: string }) & {
    status: number
    statusText: string
  }
}
```

#### Type Parameters

<ParamField type="generic" name="T">
  The success response data type.
</ParamField>

<ParamField type="generic" name="E" default="unknown">
  The error response type.
</ParamField>

<ParamField type="generic" name="Throw" default="false">
  Whether the fetch is in throw mode.
</ParamField>

#### Behavior

**When `Throw` is `false` (default):**

Returns a discriminated union:
```typescript
{ data: T; error: null } | { data: null; error: E & ErrorMetadata }
```

**When `Throw` is `true`:**

Returns `T` directly and throws `BetterFetchError` on failure.

#### Examples

```typescript
// Default mode
const { data, error } = await betterFetch("/api/users")
if (error) {
  console.error(error.status, error.statusText)
  return
}
console.log(data)

// Throw mode
try {
  const data = await betterFetch("/api/users", { throw: true })
  console.log(data)
} catch (error) {
  if (error instanceof BetterFetchError) {
    console.error(error.status, error.statusText)
  }
}
```

### FetchEsque

A type representing a fetch-compatible function.

```typescript
type FetchEsque = (
  input: string | URL | globalThis.Request,
  init?: RequestInit,
) => Promise<Response>
```

Used for custom fetch implementations:

```typescript
import { betterFetch } from "@better-fetch/fetch"
import nodeFetch from "node-fetch"

await betterFetch("/api/users", {
  customFetchImpl: nodeFetch as FetchEsque,
})
```

### Method

HTTP method types.

```typescript
type PayloadMethod = "GET" | "POST" | "PUT" | "PATCH" | "DELETE"
type NonPayloadMethod = "GET" | "HEAD" | "OPTIONS"
type Method = PayloadMethod | NonPayloadMethod
```

### CustomHeaders

Type-safe headers with common HTTP headers.

```typescript
type CustomHeaders<T extends Record<string, string> = {}> = {
  accept?: StringLiteralUnion<
    "application/json" | "text/plain" | "application/octet-stream"
  >
  "content-type"?: StringLiteralUnion<
    | "application/json"
    | "text/plain"
    | "application/x-www-form-urlencoded"
    | "multipart/form-data"
    | "application/octet-stream"
  >
  authorization?: StringLiteralUnion<`Bearer ${string}` | `Basic ${string}`>
} & T & Record<string, string | undefined>
```

Provides autocomplete for common headers while allowing custom ones:

```typescript
const headers: CustomHeaders = {
  "content-type": "application/json", // Autocompleted
  "x-custom-header": "value", // Allowed
}
```

## Authentication Types

### Auth

Union of authentication types.

```typescript
type Auth = Bearer | Basic | Custom
```

### Bearer

Bearer token authentication.

```typescript
type Bearer = {
  type: "Bearer"
  token: typeOrTypeReturning<string | undefined | Promise<string | undefined>>
}

type typeOrTypeReturning<T> = T | (() => T)
```

#### Example

```typescript
await betterFetch("/api/protected", {
  auth: {
    type: "Bearer",
    token: "your-token",
  },
})

// Or with dynamic token
await betterFetch("/api/protected", {
  auth: {
    type: "Bearer",
    token: async () => await getAccessToken(),
  },
})
```

### Basic

Basic authentication.

```typescript
type Basic = {
  type: "Basic"
  username: typeOrTypeReturning<string | undefined>
  password: typeOrTypeReturning<string | undefined>
}
```

#### Example

```typescript
await betterFetch("/api/protected", {
  auth: {
    type: "Basic",
    username: "user",
    password: "pass",
  },
})
```

### Custom

Custom authentication header.

```typescript
type Custom = {
  type: "Custom"
  prefix: typeOrTypeReturning<string | undefined>
  value: typeOrTypeReturning<string | undefined>
}
```

#### Example

```typescript
await betterFetch("/api/protected", {
  auth: {
    type: "Custom",
    prefix: "Token",
    value: "api-key-123",
  },
})
// Results in: Authorization: Token api-key-123
```

## Retry Types

### RetryOptions

```typescript
type RetryOptions = LinearRetry | ExponentialRetry | number
```

Can be:
- A number (retry attempts with default settings)
- A `LinearRetry` configuration
- An `ExponentialRetry` configuration

### LinearRetry

Fixed delay between retries.

```typescript
type LinearRetry = {
  type: "linear"
  attempts: number
  delay: number
  shouldRetry?: RetryCondition
}

type RetryCondition = (
  response: Response | null,
) => boolean | Promise<boolean>
```

#### Example

```typescript
await betterFetch("/api/users", {
  retry: {
    type: "linear",
    attempts: 3,
    delay: 1000, // 1 second between retries
    shouldRetry: (response) => response?.status === 429,
  },
})
```

### ExponentialRetry

Exponentially increasing delay between retries.

```typescript
type ExponentialRetry = {
  type: "exponential"
  attempts: number
  baseDelay: number
  maxDelay: number
  shouldRetry?: RetryCondition
}
```

#### Example

```typescript
await betterFetch("/api/users", {
  retry: {
    type: "exponential",
    attempts: 3,
    baseDelay: 1000, // Start with 1 second
    maxDelay: 10000, // Cap at 10 seconds
    shouldRetry: (response) => {
      return response?.status === 429 || (response?.status ?? 0) >= 500
    },
  },
})
// Retries with delays: 1s, 2s, 4s (capped at maxDelay)
```

## Schema Types

### Schema

API schema definition.

```typescript
type Schema = {
  schema: FetchSchemaRoutes
  config: SchemaConfig
}

type FetchSchemaRoutes = {
  [key in RouteKey]?: FetchSchema
}

type RouteKey = StringLiteralUnion<`@${Methods}/`>
type Methods = "get" | "post" | "put" | "patch" | "delete"
```

### FetchSchema

Schema for a single route.

```typescript
type FetchSchema = {
  input?: StandardSchemaV1
  output?: StandardSchemaV1
  query?: StandardSchemaV1
  params?: StandardSchemaV1<Record<string, unknown>> | undefined
  method?: Methods
  headers?: StandardSchemaV1<Record<string, unknown>>
}
```

<ParamField path="output" type="StandardSchemaV1">

### SchemaConfig

Configuration for schema.

```typescript
type SchemaConfig = {
  strict?: boolean
  prefix?: "" | (string & Record<never, never>)
  baseURL?: "" | (string & Record<never, never>)
}
```

<ParamField path="strict" type="boolean" default="false">

</ParamField>

<ParamField path="prefix" type="string">

</ParamField>

<ParamField path="baseURL" type="string">

</ParamField>

## Utility Types

### StringLiteralUnion

Provides autocomplete for string literals while allowing any string.

```typescript
type StringLiteralUnion<T extends string> = T | (string & {})
```

### Prettify

Flattens intersection types for better IDE display.

```typescript
type Prettify<T> = {
  [K in keyof T]: T[K]
} & {}
```

### StandardSchemaV1

Interface for schema validation libraries.

```typescript
interface StandardSchemaV1 {
  "~standard": {
    version: 1
    vendor: string
    validate: (value: unknown) => Result
  }
}
```

Supports: Zod, Valibot, ArkType, Yup, and other Standard Schema compatible libraries.
